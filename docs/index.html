Exodus Plugin Documentation
Basic plugin architecture

The plugin system for exodus is based around using Microsoft's COM interfaces to allow third-party components access to the GUI based events and packets coming into the application from the jabber server.

To function "as a plugin" in exodus, a developer must create an ActiveX library which implements at least the IExodusPlugin interface. This interfaces provides basic functionality which allows Exodus to control the plugin. Upon startup, the plugin is passed a reference to the main Exodus controller which allows the plugin itself to manipulate data, fire events, and send packets.

Interfaces which should be implemented by Plugins
  • IExodusPlugin
  • IExodusChatPlugin
Interfaces implemented by Exodus
  • IExodusController
  • IExodusChat
  • IExodusRoster
  • IExodusRosterItem
  • IExodusPPDB
  • IExodusPresence
